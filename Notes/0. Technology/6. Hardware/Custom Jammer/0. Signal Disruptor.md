# Jammer 

El **Jammer** o **Signal Disrupter** es un dispositivo que transmite señales de radio ([[1. Electromagnetics Waves Properties]] & [[2. Types of Electromagnetics Waves#Radio waves]]) deliberadamente para *interrumpir*, *bloquear* o *interferir* en las comunicaciones inalámrbicas. Al anular las transmisiones legítimas de las mismas frecuencias, prova una denegación de servicio en una zona concreta.

----
# Physical Layer

Al nivel *físico*, el jammer es una **interferencia intencionada**. La comunicación depende de un dispositivo receptor que sea capaz de extraer una señal útil del ruido de fondo. El parámetro fundamental es la relación *señal-ruido* (*Signal to Noise Ration* - *SNR*).

```txt
SNR = P_signal / P_noise
```

Un dispositivo receptor tiene un umbral mínimo de relación SNR por debajo del cual no puede decodificar. Este umbral varía según la modulación:

| Modulation   | Minimum SNR | Adittional Notes                     |
| ------------ | ----------- | ------------------------------------ |
| BPSK         | ~6-8 dB     | -                                    |
| QPSK         | ~10 dB      | -                                    |
| 16-QAM       | ~18 dB      | -                                    |
| LoRa SF7     | ~ -6 dB     | Spread spectrum de gran resistencia. |
| NRF24 (GFSK) | ~ -85 dBm   | Sensible                             |
El **Jammer** funciona *elevando el nivel de ruido de fondo* lo suficiente como para que la señal quede por debajo del umbral de decodificación del receptor. No es necesario que el receptor reciba una señal muy potente: *basta con que haya suficiente energía de interferencia en la frecuencia* y *tiempo* para corromper la trama.

----
# The RF Physics behind Interference ([[3. Radio Frequency Spectrum]])

Existe una *pérdida de señal en el espacio libre* (*Free Space Path Loss*), ya que, la potencia de la señal disminuye con la distancia. Esto es representado por la ecuación de Friis:

```txt
P_recceived = P_tx * G_tx * G_rx * (λ / 4πd)²
```

Eso representa que a mayor cercania del jammer con el objetivo a interferir, mayor será la ventaja de potencia. Una reducción de la distancia en un factor de 10x equivale a una ganancia de 20dB en la potencia de interferencia relativa.

Por otro lado, también existe el *nivel de ruido de fondo* (*Noise floor*), donde cada receptor tienen un nivel de ruido de fondo termico, representando como:

```txt
N = k * T * B
```

- `k` = Constante de Boltzmann's (1.38e-23)
- `T` = Temperatura en *Kelvin* (~290K ambient).
- `B` = Ancho de banda en `Hz`.

En la frecuencia de `2.4 GHz` y con un ancho de banda de `1 MHz`, el umbral de ruido térmico es de aproximadamente `-114 dBm`. Para provocar un fallo en la decodificación, *basta con que un dispositivo de interferencia eleve el nivel de interferencia a unos pocos* `dB`*por encima del nivel de la señal en la antena del receptor*.

# Jamming Strategies - (Technical Classification)

A continuación se detallan algunas de las técnicas conocidas con el fin de llevar a cabo una interferencia en un sistema de señales electromagnéticas junto a su descripción y breve funcionamiento.

1. **Continuous Wave (CW) Jamming**: Transmite una onda portadora constante en la frecuencia objetivo. Se conoce como la forma más sencilla de realizar interferencias. Tiene alta eficiencia espectral para objetivos de un solo canal. 
   
   Sin embargo, cuenta con un importante punto débil, y es que se puede filtrar fácilmente si el receptor cuenta con un filtro de muesca o agilidad de frecuencia.

1. **Spot Jamming**: Concentra toda la potencia en una sola frecuencai o banda estrecha. Ofreciendo una máxima densidad de potencia, pero solo es eficaz si el objetivo no cambia de frecuencia.

2. **Sweep Jamming**: Barre la onda portadora a lo largo de un rango de frecuencias, esto resulta eficaz contra el espectro ensanchado si la velocidad de barrido es lo suficientemente rápida como para volver a pasar por cada canal antes de que se vuelva a ensamblar el paquete.
   
   Si el barrdio cubre `80 MHz`en `1 ms`, cada canal de `1 MHz`recibe aproximadamente `12,5 µs`de inferencia por ciclo de barrido. Que eso sea suficiente depende de la duración del símbolo del protocolo.
   
2. **Reactive** / **Smart Jamming**: Este tipo de técnica de interferencia primero recibe e interpreta las ondas y solo emite interferencias cuando se detecta un paquete. Es la opción más eficiente en cuando a consumo energético. Muy eficaz contra protocolos con un ciclo de trabajo bajo. 

3. **Deauthentication** (**Protocol-Layer Jamming**): No se trata de ruido a nivel de radiofrecuencia, sino de una inyección de paquetes diseñada específicamente para provocar que la propia pila de protoclos desconecte a los usuarios pares. Las tramas de desauténticación de Wi-Fi 802.11 son el ejemplo clásico ([[Deautentication Attack]]). *Aunqué técnicamente no se trata de interferencia física*.

# Hardware Selected

Si bien los detalles de cada componente seleccionado y el por que se encuentra en la sección de [[1. Components]], es importante hacer un importante disclaimer.

> [!warning] Legalt TX limits.
> Como vimos, el setup base es:
> - x2 NRF24L01: `2.4GHz` transmisor, con canales/DR configurables.
> - x2 Amplificadores: Salida de `2W`/ `30dBm`.
> - x2 12 dBi Antenna: Antena direccional de alta ganancia.
>   
>   El amplificador alcanza los 30dBm (`1W`), y las antenas de `12 dBi` aporta ganancia direccional, lo que permitiría alcanzar un *EIRP* (*Effective Isotropic Radiated Power*) de más de `40 dBm`.
>   Sin embargo, es sumamente **IMPORTANTE** aclarar que ese nivel de potencia, supera los límites legales e transmisión en la mayoría de jurisdicciones, por lo cual es importante entender que se esta llevando a cabo únicamente con fines de estudio y laboratorio local y personal, fomentados por el trabajo.

Así mismo, el **NRF24L01** es un transmisor GFSK con importantes carácteristicas para este caso:

- *Configurable Channel*: 2400 + N `MHz` (N = 0 - 125) → Cubriendo de `2.400` - `2.525 GHz`.
- *Configurable Data Rate*: `250 kbps`, `1 Mbps`, `2 Mbps`.
- *CRC*: 1 or 2 `bytes`.
- *Auto-ACK* y *Auto-Retransmit* (Configurable).
- *6 RX* pines con direcciones individuales.

Aún así, para la aplicación en interferencia, los registros críticos son:

| **Register** | **Purpose**                |
| ------------ | -------------------------- |
| *RF_CH*      | Configurar canales (0-125) |
| *RF_SETUP*   | Data Rate + PA lvl.        |
| *CONFIG*     | TX/RX model, CRC config.   |
| *EN_AA*      | Auto-ACK por pipe.         |
| *SETUP_RETR* | Retransmit delay/count.    |
