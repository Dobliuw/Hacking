# Neural Networks

Las **redes neuronales** artficiales son modelos computacionales de inteligencia artificial inspirados en el cerebro humano. Consiste en capas de nodos (Neuronas) interconectados que procesan información, aprenden patrones complejos a partir de datos y mejora su precisión con el tiempo, siendo fundamentales para el *deep learning* en tareas como recnocimiento de imágenes y voz.

---
# Perceptron

El **perceptrón** es la unidad computacional más básica de red neuronal artificial, diseñada para la clasificación binaria (Decidir entre dos categorias). Recibe entradas, las pondera, las suma y aplica una *función de activación* para producir una salida. Las *deep networks* simplementen apilan múltiples capas de perceptrones.
![[Pasted image 20260326145927.png]]

Las funciones de activación son esenciales porque introducen no linealidad. *ReLU* (Rectified Linear Unit) domina en capas ocultas por ser computacionalmente barata y evitar el problema del gradiente que se desvanece. *Sigmoid* y *Tanh* se usan en capas de salida o en arquitecturas recurrentes. *Softmax* es el estándar para clasificación multiclase porque convierte un vector de puntajes en una distribución de probabilidad.

# Deep Networks

Una **red profunda** apila múltiples capas de *perceptrones*. La profundidad permite aprender *representaciones jerárquicas*: Las capas trempranas aprenden features simples y las capas tardías las combinan en conceptos abstractos.

![[Pasted image 20260326181641.png]]

El proceso de entrenamiento tiene dos fases. En el *forward pass*, los datos se propagan capa a capa hasta producir una predicción `ŷ`. Se calcula la pérdida comparando `ŷ` con la etiqueta real `y`. En el *backward pass* (*Backpropagación*), se aplica la regla de la cadena del cálculo diferencial para calcular `∂L/∂w`, para cada peso de la red, propagando el error desde la salida hacia la entrada. Finalmente, un optimizado (SGD, Adam, etc.) actualiza los pesos.

> [!warning] Problems
> - *Sobreajuste*: En este caso la red memoriza los datos de entrenamiento en lugar de generalizar.
> - *Subajuste*: La red no tiene suficiente capacidad para aprender el patrón
>   
>   Ante estos problemas, las herramientas para combatirlos incluyen la regularización L1/L2, *dropout* (Apagar neuronas aleatorias durante el entrenamiento) y *batch normalization* (Normalizar las actividades entre capas para estabilizar el entrenamiento).

# CNNs 

Las **Convolutional Neural Networks** son redes diseñadas para datos con estructura espacial (Imágenes). En lugar de conectar cada píxel con cada neurona, usan *filtros* que se deslizan por la imagen detectando patrones locales.

![[Pasted image 20260326182432.png]]

Conceptos de estas tocan:

- *Filtro* / *Kernel*: Pequeña matriz (Ej. 3x3) que se desliza por la imagen haciendo productos punto. cada filtro aprende a detectar un patrón.
- *Feature map*: El resultado de aplicar un filtro a la imagen.
- *Pooling*: Reduce la resolución espacial, haciendo el modelo más eficiente y robusto a traslaciones.
- *Weigth Comparation*: El mismo filtro se usa en toda la imagen (Teniendo muchos menos parámetros qeu una red densa).

![[Pasted image 20260326182757.png]]

Básicamente una convulción multiplica elemento a elemento el kernel con una región de la imagen y suma los resultados, produciendo un escalar en el feature map. El kernel se desliza por t oda la imagen con un *stride* (Paso). Si se usan 64 filtros distintos, se obtienen 64 feature maps, cada uno respondiendo a un patrón diferente. El *receptive field* de una neurona profunda es grande aunque su conexión directa sea solo una región pequeña: cada cpaa extiende la "vista indirectamente".

El *max pooling* toma el máximo de una región (Típicamente 2x2), reduciendo la resolución a la mitad. Esto aporta invariancia a pequeñas traslaciones y reduce el costo computacional. Arquitecturas modernas como *ResNet* añaden conexiones residuales para poder entrenar redes de cientes de capas.

# RNNs & LSTMs

Las **Recurrent Neural Networks** (**RNNs**) procesan datos secuenciales (Texto, audio, series temporales) manteniendo un *estado oculto* que actúa como memoria. Sin embargo, sufren de "*memoria a corto plazo*" y desvanecimiento gradiente.

En cambio las **Long Short-Term Memories** resuelven este problema de las **RNN** con puertas que controlan qué recordar y qué olvidar. Han sido diseñadas para aprender dependencias a largo plazo en datos secuenciales. 

Su estructura interna incluye "compuertas" (*Input*, *Forget* y *Output*) que gestionan qué información almacenar, olvidar o actualizar en una "celda de estado".

![[Pasted image 20260326190255.png]]

Conceptos claves:

- *Estado oculto* (`h(t)`): La "memoria" que se pasa de un paso al siguiente.
- *Problema del gradiente que se desvanece*: En secuencias largas, la señal de error se hace tan pequeña que la red no aprende dependencias distantes.
- *Forget gate*: Decide qué parte de la memoria a largo plazo descartar.
- *Input gate*: Decide qué información nueva agregar al estado de celda.
- *Output gate*: Determina qué parte del estado de celsa se convierte en salida.

# Transformers

Los **transformers** eliminaron completamente la recurrencia. La ideal central fue, en lugar de procesar *tokens* secuencialmente y confiar en que el estado oculto "recuerde", dejamos que cada token atienda directamente a todos los demás en una sola operación matricial.

![[Pasted image 20260326190838.png]]

El mecanismo de *self-attention* funciona de manera tal que cada token genera tres vectores mediatne multiplicación por matricies aprendidas, la *query* (Que búsca este token), la *key* (Qué "etiqueta" representa) y el *value* (Qué información aporta). Para calcular la salida de un token, se computa el producto punto entre su query y todas las keys, se divice por `√dₖ` para estabilizar los gradientes, se aplica *softmax* para obtener una distribución de probabilidad (Los pesos de atención), y finalmente se calcula la medida ponderada de todos los values.

El *multi-head attention* ejecuta este proceso en paralelo `h`veces, con distintas matrices de proyección, permitiendo que la red capture diferentes tipos de relaciones simultáneamente (Correferencia, dependencia sintáctica, semántica, etc.). Las salidas de todas las cabezas se concatenan y se proyectan.

Dos componentes estructurales son críticos. Las *conexiones residuales* (`x + Sublayer(x)`) permiten que el gradiente fluya directamente a capas inferiores, haciendo posible entrenar redes de decenas de capas. La *layer normalization* estabiliza la distribución de activaciones en cada capa, acelerando el entrenamiento. El bloque completo se apila N veces.

Finalmente, el *positional encoding* es necesario porque la atención no tiene noción de orde: un token en la posición 1 y uno en la posición 100 son indistinguibles sin esta señal adicional. Se añade un vector sinusoideal de frecuencias distintas a cada *embedding*, codificando la posición de forma que el modelo pueda parender relaciones de distancia relativa.
