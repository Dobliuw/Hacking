# Components

Un sistema de **Machine Learning** no es solo el modelo, como suele creerse, si no más bien un *pipeline completo de ingeniería de datos*, *entrenamiento*, *evaluación* y *operación*. En la práctica esto se conoce como **ML Lifecycle** o **ML Pipeline**, y es lo que da origen al campo de *MLOps*.

#### Data

Los **datos** representan la *experiencia a partir de la cual el sistema aprende*. En machine learning no se programa explícitamente el comportamiento del sistema; en cambio, el modelo intenta *inferir patrones estadísticos* a partir de ejemplos observados.

Los datasets suelen contener muchos pares de datos que sirven como ejemplos de una relación. Esto implica que los datos son una *representación parcial del mundo real*. Nunca acpturan toda la complejidad del fenómeno, sino una muestra limitada y condicionada por cómo se recolectaron.

Por esto, entender de donde vienen los datos es tan importante como el modelo. Existen varias fuentes típicas:

- **Datos generados por sistemas**: Logs de servidores, registros de sensores, telemetría de aplicaciones, transacciones financieras, etc.
- **Datos recolectados manualmente**: Encuenstas, etiquetado humano, anotaciones para visión o lenguaje.
- **Datos derivados**: Resultados de otros sistemas o modelos previos.
- **Datos públicos o abiertos**: Datasets académicos o corporativos.

En muchos proyectos, esta fase, con el fin de obtener *datos confiables* suele ser la más costosa. Esto implica diseñar pipelines de recolección, sistemas de almacenamiento y mecanismos de validación de dichos datos. 

Conceptualmente, los datos suelen dividirse en dos tipos:

- **Features**: Variables que describen la situación o el objeto que queremos analizar.
- **Labels**: La respuesta o resultado que el modelo debe aprender a predecir.

Un aspecto crítico es que los datos deben representar la *distribución real del problema*. Si el dataset no refleja adecuadamente el mundo donde el modelo se usará, el aprendizaje será incorrecto. Este fenómeno se conoce como **dataset bias** o **distribution mismatch**. Además del problema estructural: Los datos crudos casi nunca están listos para ser usados directamente. Pueden contener errores, incosistencias o estructuras incompatibles con los algoritmos, debido a esto existe la siguiente fase.

#### Pre-Procesing

El **preprocesamiento** es el proceso mediante el cual los *datos curos se transforman en representaciones adecuadas para el aprendizaje automático*. Esta fase es necesaria porque los algoritmos de ML operan sobre estructuras numéricas bien definidas, mientras que los datos reales suelen ser heterogéneso y desordenados.

El preprocesamiento cumple varios objetivos:

1. *Garantizar la calidad de los datos*: Esto implica identificar valores faltantes, corregir incosistencias y eliminar registros corruptos. En datasets grandes es común encontrar duplicados, errores de medición o valores imposibles.

2. *Representar la información de manera útil para el modelo*: Esto se conoce como **feature engineering** y consisten en transformar variables originales en nuevas variables que captures mejor las relaciones relevantes. ejemplo, un timestamp bruto puede transformarse en variables como hora del día, día de la semanao estacionalidad. Estas transformaciones permiten que el modelo capture patrones que de otro modo quedarían ocultos.

3. *Convertir datos no númericos en representacioens matemáticas*: Los algoritmos de aprendizaje trabajan con vectores numéricos, por lo que variables categóricas o textuales deben transformarse mediante técnicas como codificación categórica o embddings.

4. *Normalizar o escalar variables*: Muchas veces una variable tiene valores entre 0 y 1 y otra entre 0 y 10.000, algunos algoritmos pueden volverse inestables o dar más peso a una de ella simplemente por su escala.

5. *Separación de dataset en subconjuntos independientes*: Esto es crucial porque el modelo debe evaluarse con datos que no haya visto durante el entrenamiento. Si se entrenara y evaluara con los mismos datos, el sistema simplemente memorizaría los ejemplos sin aprender aptrones generalizables.

Por todo esto, los dataset suelen dividirse en tres partes conceptuales:

- **Datos de entrenamiento**: Utilizados para ajustar el modelo.
- **Datosde validación**: Utilizados apra ajustar hiperparámetros y seleccionar modelos.
- **Datos de prueba**: Utilizados para estimar el rendimiento final
