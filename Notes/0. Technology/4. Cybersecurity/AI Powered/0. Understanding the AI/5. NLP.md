# Natural Language Processing

El **NLP** (**Natural Language Processing**) es una *rama de la inteligencia artificial* que capacita a las máquinas para entender, interpretar, analiar y generar el lenguaje humano, ya sea escrito o hablado. Combina la lingüística computacional con el aprendizaje automático para *procesador datos lingüísticos complejos y naturales*.
# Text, Tokenization & Embeddings

El primer desafío del NLP es convertir texto (Secuencia de carcateres) en *números* que una red pueda procesar. Este pipeline tiene tres etapas:

1. *Tokenizar*.
2. *Asignar IDs*.
3. *Proyectar a un espacio vectorial continuo*.
###### Tokenization

La **tokenización** es el proceso fundamental de dividir texto (Palabras, frases, caracteres, etc.) en *unidades más pequeñas* llamadas `tokens` para que los modelos de lenguaje (LLM) puedan procesarlos. Estos tokens, que pueden ser palabras completas o parte de ellas, se convierten en números para que la AI entienda y genere lenguaje.

La **tokenización BPE** (Byte Pair Encoding) es el estándar moderno, comienza con caracteres individuales y fusiona iterativamente los pares más frecuentes hasta alcanzar un tamaño de vocuabulario objetivo. La ventaja clave es que nuncap roduce tokens desconocidos (OOV): cualquier palabra nueva se descompone en subpalabras conocidas.

###### Embeddings

Los **embeddings** son *representaciones numéricas de datos* (Palabras, imágenes, audio) en forma de *vectores multidimensionales* que capturan su significado *semántico*. De esta manera, transforman información compleja en listas de números donde elementos similares se sitúan cercanos entre sí en un espacio vectorial, permitiendo a la IA comprender contextos y relaciones.

![[Pasted image 20260326195340.png]]

Los **embeddings estáticos** (Word2Vec, GloVec) aprenden un único vector por palabra sin importar el contexto. Los **embeddings conextuales** (Que salen de las capas de un *transformer* - [[4. Neural Networks#Transformers]]), son distintos para cada aparición "banco" en "fui al banco a sacar dinero" y "me senté en el banco del parque" producen vectores diferentes, capturando la desambiguación.

# Language Model

Al final, los **modelos de lenguaje** son un tipo de software que usa técnicas de aprendizaje, como *machine learning* ([[2. Machine Learning System Components]]) con el fin de aprender a asignar probabilidades a secuencias de texto.

La diferencia arquitectónica entre *GPT* y *BERT* es fundamental.

- **GPT** usa *atención causal* (*Masked*): Cada token solo puede atender a los anteriores, lo que hace apto para generación pero limita la comprensión bidireccional.
- **BERT**: usa *atención bidireccional completa*: Ideal para tareas de comprensión donde se tiene acceso a todo el contexto. 

La *perplejidad* mide cuánto "le sorprende" al modelo cada token nuevo; un modelo perfecto que siempre predice el token correcto tendría `PPL=1`.
